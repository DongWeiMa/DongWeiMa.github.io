debug模式下加入json schema–模型的替代

没有可视化的模型,导致接口的返回数据有时候少字段,或者自己加了字段过段时间也不清楚了.
json schema能帮我自动记忆和校验我是否少了字段,并且可以作为文档给前端

mock 数据自动更新,采集线上运行的数据,线上数据有时候有问题,也会自动发邮件报警.

java library
https://github.com/everit-org/json-schema#when-to-use-this-library

简单两个要求
1.告诉我返回的数据哪些点不符合要求就可以.
2.数据可以来自数据库

本来是postman 中写脚本,但是一般也懒得测试,而且postman不属于团队.

schema生成:
http://jsonschema.net/#/
schema 语法
https://json-schema.org/understanding-json-schema/reference/boolean.html#boolean

直接从数据库中读取/读取文本

接口mock:
sss/mock
接口正式:sss.com
接口测试:sss/debug

url是
/v1/79/course/approver/list?page=1&pageSize=10&corpId=wx981d0202130ed8e7
比如mock地址
bridge.api.xiaoyuanhao.com/mock/v1/79/course/approver/list?page=1&pageSize=10&corpId=wx981d0202130ed8e7

现访问mock吧
rap只配置一个url,文档在mock或测试环境的返回参数中有
返回参数如果有errors不为空,说明接口有问题

有问题直接给requestId和errors就可以
