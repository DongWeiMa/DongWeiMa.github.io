sql template,
同步配置,回退配置

1.压测压力上不去的问题解决
2.调课,安吉督导的接口提供
3.触发器记录数据库的数据集相关数据的修改,保留操作记录和数据的历史版本.
4.接口提供debug模式和mock模式
debug模式提供
1).json schema—文档说明,返回数据校验,弥补无可视化结构的缺点.返回的校验会详细解释返回数据错在哪里
2).返回数据添加requestid,并且直接返回链接,可以直接查到鹊桥的该接口日志
mock模式提供mock数据
debug模式,mock模式,一般模式,前端的所有接口只需要改域名.和以前的开发模式完全兼容.
解决需要去rap重复定义的缺点.
之后加上离散采集数据,根据线上接口的返回自动设置mock数据和jsonSchema.
5.设计半实现sql template 执行工具.
没有sql template指定工具的情况下,流程是这样的
1).找到数据集,找出模版sql
2).复制sql到数据库,
3).替换参数(时间长,参数还需要去找起来)
4).调试sql
5).将修改完的sql再改回模版sql
6).更新数据集
7).清除缓存.
有了sql template工具流程是这样的
1.找到数据集id
2.加载模版sql和参数
3.调试模版sql
4.直接更新数据集
优点:缓存参数,自动渲染sql,不需要手动修改模版sql中的参数
甚至没有参数的情况下,可以加上离散采集数据,根据线上接口的参数自动设置,下一次调试就可以直接用了
6.利用飞冰,编写sql template页面,鹊桥后台管理系统的一些展示页面;用鹊桥配置查询接口
7.加强鹊桥的url约束,不再像之前那样,前端可以随意定义;修复一些测出来的bug
