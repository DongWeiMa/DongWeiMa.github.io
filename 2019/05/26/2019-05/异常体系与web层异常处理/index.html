异常体系与web层异常处理

May 26, 2019 | spring

对于异常的处理,要结合日志的理解.

1.异常至少应该分为两个大类,一种是不可恢复异常,一种可自恢复异常.
2.可恢复异常中又分为影响业务正确性和不影响业务正确性

据此定了两个根异常
BusinessException:该异常系统可以自恢复,也不会响应系统的业务正确性.–比如鉴权失败,参数校验失败等.
ErrorException: 该异常表明,要不系统没办法自恢复,要不就是会影响业务的正确性.比如数据库连不上了,比如支付成功的消息没有及时发出去等.

其余自定义异常就定义为枚举类,传入BusinessException或ErrorException的构造器生成异常,而不是定义一大堆自定义异常.每个异常都需要定义自己的msg信息和error_code.这个通过LocalError接口约束.

将异常分为这两类之后,基于error级别日志的异常报警也就能做了.
