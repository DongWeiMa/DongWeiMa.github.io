sql注入问题解决

1.原本采用预编译的方式,但是遇到很多问题
1)预编译的时候,类型必须匹配,这一点很恶心.临时采用了数字自动转字符串,但是一些奇葩的数据就比较奇怪,比如pt,是数字,但是数据库是字符串

2)集合参数处理, in的时候,集合是用()处理的,当时pg的array类型,就变为了’{,,,}’的方式

后来改了方案,不采用预编译的方式,改为了转义

1.Druid 的wall filter
https://github.com/alibaba/druid/wiki/%E5%B8%B8%E8%A7%81%E9%97%AE%E9%A2%98
2.StringEscapeUtils类的转义与反转义方法
org.apache.commons.lang.StringEscapeUtils
类可以对js sql html xml等代码进行转义!用来处理前端js注入代码
SQL特殊字符转义
3.参数检测
但是在每个方法中都使用手工编写检测逻辑的方式并不是一个好主意。阅读 Spring 源码，您会发现 Spring 采用一个 org.springframework.util.Assert 通用类完成这一任务。
