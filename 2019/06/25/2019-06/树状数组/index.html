树状数组

Jun 25, 2019 | 算法

解决的问题

先来知道树状数组是用来干嘛的–快速求线性数据结构的前缀和.

那么问题来了,前缀和? 预处理s[i] = 前i个元素总和,
t[i,j]=s[i]-s[j+1] 不就可以快速求了吗?

但是,现实中,线性数据结构中的元素不是一直不变了,那么,当他变化的时候,要重新求s数组,那么时间复杂度就退化到O(N)了.

树状数组可以在变化的情况下,仍然保持时间复杂度为O(log2(N)).
那么他是怎么做到的呢?

原理

树状数组是Peter M. Fenwick在类比整数与二进制的关系之后灵感中产生的.
我们先来看整数与二进制的关系的样例.
13 = 1+4+8 = (1101)2
在求13 这个值得时候,我们可以拆分为三个节点的值.
任何整数转换成二进制只需要最多log2(N)个节点求和就行.基本就是分治的思想.

那么线性数据结构如果也看成类似二进制的这种形式,那么时间复杂度不就是log2(N).
那么Peter M. Fenwick 是怎么做的呢?
分段求和.
首先将N转化为2进制.
N = 2^k1+2^k2…+2^ki
k1<k2<..<ki
我们怎么分段求和呢?
0-2^k1 为一段
2^k1+1 + 2^k2 为一段
…
(2^(i-1))+1 + 2^i 为一段

那么在求N的前缀和就只需要将N转化为二进制,根据1的分布求和即可.

这个时候我们再去看树状数组的一些定义

比如我们假设用C[i]表示f[1]…f[i]的和，而用tree[idx]表示子序列.按照定义实际上tree[idx]是那些indexes from (idx - 2^r + 1) to idx的f[index]的和,其中r是idx最右边的那个非零位到右边末尾的0的个数。

tree[13] = f[13]
tree[12] = f[12]+f[11]+f[10]+f[9]
tree[8] = f[8]+….f[1]
C1 = A1
C2 = A1 + A2
C3 = A3
C4 = A1 + A2 + A3 + A4
C5 = A5
C6 = A5 + A6
C7 = A7
C8 = A1 + A2 + A3 + A4 + A5 + A6 + A7 + A8
…
C16 = A1 + A2 + A3 + A4 + A5 + A6 + A7 + A8 + A9 + A10 + A11 + A12 + A13 + A14 + A15 + A16
这里有一个有趣的性质，下午推了一下发现：
设节点编号为x，那么这个节点管辖的区间为2^k（其中k为x二进制末尾0的个数）个元素。因为这个区间最后一个元素必然为Ax，
所以很明显：Cn = A(n – 2^k + 1) + … + An

到此,类比就结束了.

理解之后,其他的更新,查询就比较顺其自然了.
主要是两个函数:
前缀和的计算–路径求和
更新的维护–只需要更新所在路径上节点即可.

基本操作是计算整数里最右边的那个非零位
在Peter M. Fenwick的论文中提到了三种方法。
return x-(x&(x–1));
return x&(-x);
return x&(2^k-x);2^k is a power of 2 greater than the table size.

以上是s[i]的求解,其他的s[i,j]做差即可
